# Runtime

The runtime is the core execution engine for AI Employees.

**Principles:**
- Does not contain business logic or API routes (these belong in `backend`).
- Never directly queries the `database`.
- Consumes configurations generated by `backend/services`.
- Responsible for planning, memory retrieval, execution, telemetry, and workflow orchestration.

**Employees as Configurations:**
Employees are not hardcoded Python classes. They are configurations detailing prompt, tools, memory, etc., executed by the runtime workflow engine.
